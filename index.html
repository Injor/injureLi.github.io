RPC远程调用

Posted on 2020-06-10 In Java

要解决的问题

  • 协议约定问题
    • 如何规定远程调用的语法？
    • 参数如何传递，什么时候是头，什么时候是尾？
    • 如何表示数据？
  • 传输协议问题
    • 发生错误、重传、丢包、性能问题怎么办？
  • 服务发现问题
    • 一个服务实现了哪些远程调用，从哪个端口可以访问？

协议类型

文本类 二进制类
SOAP Hessian2
RESTful Protocol Buffers
  • 基于XML的SOAP协议（Simple Object Access Protocol）全称简单对象访问协议，设计是面向动作的，因而往往因为架构问题导致并发量上不去。
  • 基于JSON的RESTful协议（Representational State Transfer）表述性状态转移，是一种架构模式，主要面向资源，提供无状态服务，有利于横向扩展应对高并发。
  • 基于二进制的RPC协议Hessian2，不需要定义协议文件，是自描述的
  • 基于二进制的GRPC协议Protocol Buffers，需要写协议文件。网络传输使用 HTTP 2.0，服务治理可以使用基于 Envoy 的 Service Mesh。

二者对比

Dubbo

  • Dubbo中默认的RPC协议是Hessian2。为了保证传输的效率，Hessian2 将远程调用序列化为二进制进行传输，并且可以进行一定的压缩。
  • Hessian2是自描述的，不需要再定义协议文件
  • Dubbo使用Netty的网络传输框架，基于Socket实现高性能服务端
  • 还提供其他高性能序列化方式，例如Dubbox 从 Spark 那里借鉴 Kryo
  • 需要客户端和服务端共享jar，通过接口的方式相互依赖，强依赖关系，需要严格的版本控制，对程序无入侵

OpenFeign

  • 消费端和提供端可以不用共享JAR，无接口依赖，定义好相关的json字段即可，对程序有一定入侵性
  • 程序与文本之间转换效率低，不适合内部大数据交换
  • 相比于二进制RPC，传输需要的带宽更大

API网关

Posted on 2020-06-09 Edited on 2020-06-10 In Java

职能

  • 请求接入
  • 业务聚合
    • 下游服务注册及管理
    • API聚合与数据编排，针对API的定制化、复用
  • 中介策略。实现鉴权、反爬、路由、分流、流控、降级等策略
    • 基础安全保障，侧重应用层安保能力建设
    • 流量治理，侧重于流量隔离与调度
  • 统一管理。提供配置管理平台，对所有API服务的调用生命周期和中介策略进行统一管理

分类

  • 流量网关。与后端业务无关，侧重于全局流量的安全与稳定
  • 业务网关。针对某个业务的流控，下流多API聚合等

定位

作为网关，最重要的是安全稳定，其次是降本增效，如何在保证安全稳定的前提下，以最快的速度帮助业务达成，这就是API网关的定位

实现

Kong

Kong是一个在Nginx运行的Lua应用程序，由lua-nginx-module实现。Kong和OpenResty一起打包发行，其中已经包含了lua-nginx-module。OpenResty不是Nginx的分支，而是一组扩展其功能的模块。

Tyk

Tyk是一个开源的、轻量级的、快速可伸缩的 API网关，支持配额和速度限制，支持认证和数据分析，支持多用户多组织，提供全 RESTful API。基于 go 编写。

Traefik

Traefik 是一个现代 HTTP 反向代理和负载均衡器，可以轻松部署微服务，Traeffik 可以与您现有的组件（Docker、Swarm，Kubernetes，Marathon，Consul，Etcd，…）集成，并自动动态配置。

Ambassador

Ambassador 是一个开源的微服务 API 网关，建立在 Envoy 代理之上，为用户的多个团队快速发布，监控和更新提供支持，支持处理 Kubernetes ingress controller 和负载均衡等功能，可以与 Istio 无缝集成。

Netflix Zuul

Zuul 是一种提供动态路由、监视、弹性、安全性等功能的边缘服务。Zuul 是 Netflix 出品的一个基于 JVM 路由和服务端的负载均衡器。

SpringCloud Gateway

Gateway is built on Spring Framework 5, Project Reactor and Spring Boot 2 using non-blocking APIs. Websockets are supported and it’s a much better developer experience since it’s tightly integrated with Spring.
